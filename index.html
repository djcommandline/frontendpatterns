STYLE > SIZE > ACTION > POSITION > PROPERTIES

Prototype elements using modular class definitions.

There are several school of thoughs on using non-semantic class names. From a prototyping stand point, translating style properties to class names allow for the quick application of styles to elements without having to write css rules to specific divs.

Performance arguments could be made against this approach, but unless your site and buisness requires the most effiecient usage of markup and performance it doesn't make much sense to not employ fast prototyping techniques. Especially if you are working on unproven business models or a a design/interface team that changes the game plan at regular intervals.

For most small business sites or low traffic sites it just doesn't makes sense. You can consolidate and optimze css rules and markup down the road when there is a finianical reason to do it.
